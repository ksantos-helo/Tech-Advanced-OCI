# ☁️ Resumo OCI (Oracle Cloud Infrastructure)

## 🚧 Desafios do ambiente On-Premises

Infraestrutura cara e inflexível

Limitações de:

espaço

processamento

memória

Escalabilidade difícil (troca de hardware)

Alta latência

Custos imprevisíveis

## ✅ Vantagens da OCI

💰 Previsibilidade de custos

⚡ Baixa latência

🔄 Escalabilidade (scale up/down)

🔥 Ajuste de recursos “a quente”

⚖️ Load Balancer

Flexibilidade na infraestrutura

## 🌎 Regiões (Regions)

Área geográfica com 1 ou mais Data Centers

No Brasil:

🇧🇷 São Paulo

🇧🇷 Vinhedo

## 🏢 Domínios de Disponibilidade (AD - Availability Domains)

Data centers independentes dentro de uma região

Isolamento de:

energia

rede

resfriamento

Garantem alta disponibilidade

⚙️ Domínios de Falha (Fault Domains)

Cada AD possui 3 Fault Domains

Agrupamento de hardware independente

Protege contra:

falhas físicas

manutenção

queda de energia

Permite distribuição de recursos para tolerância a falhas

📌 Exemplo:

Servidor principal (FD1)

Servidor secundário (FD2)

→ Se um falhar, o outro assume

## 🔗 Oracle Multicloud

Integração com:

AWS

Azure

Google Cloud

Uso combinado de serviços (melhor de cada nuvem)

Conexão segura via intercloud

Exemplo:

OCI (compute + MySQL HeatWave)

Azure/AWS (outros serviços)
