## ✅ 🔰 Primeiros passos 

Na sua imagem já aparece:

Método: GET ✔️
Campo de URL: “Enter URL or paste text” ✔️
Botão Send ✔️

👉 Ou seja: você já está pronta pra testar uma API!

## 🌐 🧪 Teste com a API ViaCEP (igual ao exercício)
### 1️⃣ Escolha o método

Deixe selecionado:
👉 GET (já está correto na sua tela)

### 2️⃣ Cole a URL

Copie e cole isso na barra:
```
https://viacep.com.br/ws/04101300/json/
```

### 3️⃣ Clique em Send

👉 Botão azul à direita

### 4️⃣ Veja o resultado

Se tudo der certo, vai aparecer abaixo:

📦 Body → com dados em JSON (endereço)
✅ Status: 200 → deu certo
⏱️ Tempo da requisição

## 🔍 O que você acabou de fazer (explicação simples)

Você usou:

🌐 HTTP
📥 Método GET → buscar dados
🔗 Uma API (ViaCEP)

👉 Traduzindo:
Você pediu: “me dê o endereço desse CEP”
E a API respondeu com os dados.

## ⚠️ Teste de erro (parte do exercício)

Agora tente isso:
```
https://viacep.com.br/ws/7777777/json/
```

👉 Clique em Send

Você vai ver:

❌ Status: 400
❌ Erro (CEP inválido)

👉 Isso mostra como APIs respondem quando algo está errado.

## 🧠 Resumo para estudo (bem direto)
GET → buscar dados

POST → enviar dados

URL → onde está o recurso

Send → faz a requisição

Status 200 → sucesso

Status 400 → erro na requisição
