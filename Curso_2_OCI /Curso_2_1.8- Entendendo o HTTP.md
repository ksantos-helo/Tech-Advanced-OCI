## 🌐 DevOps e Comunicação HTTP (Resumo)

Em **DevOps**, o objetivo vai além de entregar uma solução funcional: buscamos que ela funcione bem durante **todo o ciclo de vida**, com monitoramento contínuo e cuidado da infraestrutura.

---

### 🔹 Entendendo o HTTP

- O **HTTP** opera sobre a **camada TCP**, garantindo entrega confiável das requisições entre cliente e servidor.
- É essencial conhecer redes para **otimizar, manter e diagnosticar a infraestrutura**.

---

### 🔹 Estrutura de uma aplicação web

- **Front-end:** interface e interação com o usuário
- **Back-end:** lógica de negócios e processamento de dados
- Comunicação entre cliente e servidores é feita pelo **HTTP** através da rede.

---

### 🔹 Fluxo de comunicação

1. Usuário digita endereço no navegador
2. Navegador envia requisição ao **servidor front-end**
3. Front-end processa e retorna dados para renderizar a página
4. Front-end pode enviar requisições ao **back-end** para obter dados adicionais
5. Resposta do back-end é usada para popular a página no navegador

---

### 🔹 Observações importantes

- Mensagens HTTP seguem **protocolos padronizados**
- Todos os dispositivos da rede conseguem enviar, receber e interpretar essas mensagens de forma confiável
- Esse padrão permite **interoperabilidade e comunicação eficiente** entre cliente e servidor

---

### ✅ Conclusão

O acesso a uma página web envolve **uma troca estruturada de mensagens HTTP**, suportada por toda a infraestrutura de rede, entre o navegador do usuário e os servidores front-end e back-end.
