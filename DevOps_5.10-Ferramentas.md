# 🚀 Resumo de Ferramentas DevOps (Guia Rápido)

## 🔹 CI/CD (Integração e Entrega Contínua)

Automatizam o processo de build, teste e deploy de aplicações.

🛠️ GitHub Actions → Cria pipelines automatizados direto no repositório

🛠️ GitLab CI/CD → Integra versionamento + automação em uma única plataforma

🛠️ Jenkins → Ferramenta robusta e altamente configurável para automação

🛠️ AWS CodeDeploy → Automatiza deploys dentro da AWS

📌 Objetivo: reduzir erros manuais e acelerar entregas

## 🔹 Containers e Orquestração

Permitem empacotar e executar aplicações de forma padronizada.

🛠️ Docker → Cria containers (ambientes isolados e replicáveis)

🛠️ Kubernetes → Orquestra containers (escala, gerencia e distribui)

🛠️ Amazon ECS → Orquestrador de containers mais simples na AWS

🛠️ Amazon EKS → Kubernetes gerenciado pela AWS

📌 Objetivo: garantir consistência entre ambientes (dev, teste, produção)

## 🔹 Cloud (Computação em Nuvem)

Fornecem infraestrutura sob demanda.

🛠️ AWS → Líder de mercado, grande variedade de serviços

🛠️ Azure → Forte integração com ferramentas Microsoft

🛠️ GCP → Destaque em dados, IA e Big Data

📌 Objetivo: escalar sistemas sem depender de infraestrutura física

## 🔹 Infraestrutura como Código (IaC)

Permite criar e gerenciar infraestrutura via código.

🛠️ Terraform → Cria infraestrutura em múltiplas clouds

🛠️ Ansible → Automatiza configuração de servidores

📌 Objetivo: padronização, versionamento e automação da infraestrutura

## 🔹 Testes

Garantem qualidade do código antes do deploy.

🛠️ JUnit → Testes para aplicações Java

🛠️ PHPUnit → Testes para aplicações PHP

📌 Objetivo: detectar erros antes de ir para produção

## 🔹 Qualidade de Código

Analisa o código para identificar problemas.

🛠️ SonarQube → Análise estática de código

🛠️ SonarCloud → Versão em nuvem do Sonar

📌 Objetivo: melhorar qualidade, segurança e manutenibilidade

## 🔹 Versionamento de Código

Controla alterações no código.

🛠️ GitHub → Repositórios + colaboração

🛠️ GitLab → Repositório + CI/CD integrado

📌 Objetivo: rastrear mudanças e colaborar em equipe

## 💡 Conceitos-chave

💡 CI/CD → Automatização do ciclo de desenvolvimento

💡 Containers → Portabilidade de aplicações

💡 IaC → Infraestrutura como código

💡 Microserviços → Sistemas divididos em pequenos serviços

💡 YAML → Linguagem usada para configurar pipelines

## 🎯 Resumo final

DevOps é a integração de:
➡️ Desenvolvimento + Infraestrutura + Automação

Com foco em:
✔️ Entregas rápidas
✔️ Menos erros
✔️ Escalabilidade
✔️ Eficiência operacional
